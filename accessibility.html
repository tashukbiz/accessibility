Accessibility statement

I audit other people’s sites, so it’s only fair to hold my own to the same standard. This page explains what I’ve done and where the limits are.

Target standard

This website aims to meet WCAG 2.2 Level AA. Accessibility isn’t a one-time tick, so I treat this as ongoing rather than “finished”.

How this site is built

  • Semantic HTML with clear landmarks (header, nav, main, footer) and one <h1> per page.
  • A visible “Skip to main content” link that moves focus to the main region.
  • A clear, visible focus indicator on every interactive element, the exact thing missing from many sites I test.
  • Full keyboard operability. There are no custom widgets or keyboard traps; navigation is plain links that work without JavaScript.
  • Text and interface colours chosen to meet AA contrast; layout reflows to narrow screens and to 200% zoom.
  • Reduced-motion support for anyone who prefers less animation.
  • The current page is marked in the navigation with aria-current so screen-reader users know where they are.

How it was tested

Keyboard-only navigation, VoiceOver on macOS and iOS, colour-contrast checks, zoom and text-spacing checks, and automated tools (axe, WAVE, Lighthouse) as a starting point. I invite you to test it too: try tabbing through any page and watch the focus indicator move.

Known limitations

The downloadable reports (PDF and Excel) are working documents and may not yet be fully tagged for accessibility. I’m happy to provide the same information in another format on request. If you find anything on this site that doesn’t work for you, I’d genuinely like to know.
